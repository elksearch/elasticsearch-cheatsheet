## 🧱 ES|QL Anatomy — Pipe-Based Query

```
FROM source-index
| WHERE condition
| EVAL new_field = expression
| STATS agg = AGGREGATION() BY group_field
| SORT field DESC
| LIMIT 100
```

Every `|` passes output of previous command as input to next.
Think of it like Linux pipes — `cat | grep | awk | sort | head`.

```esql
// Example
FROM logs-*
| STATS
    total = COUNT(*),
    avg_duration = AVG(duration_ms),
    p95 = PERCENTILE(duration_ms, 95),
    errors = COUNT(*) WHERE level == "ERROR"
  BY service.name
| SORT total DESC
```

## ⚡ Quick Reference — Most Used in Observability

```esql
-- Error rate by service, last 1 hour
FROM logs-*
| WHERE @timestamp >= NOW() - 1 HOUR
| STATS
    total = COUNT(*),
    errors = COUNT(*) WHERE level == "ERROR",
    p95_latency = PERCENTILE(duration_ms, 95)
  BY service.name
| EVAL error_rate = ROUND(errors / total * 100, 2)
| SORT error_rate DESC

-- Top 10 slowest endpoints
FROM logs-*
| WHERE @timestamp >= NOW() - 24 HOURS
| STATS avg_ms = AVG(duration_ms), calls = COUNT(*) BY url.path
| WHERE calls > 100
| SORT avg_ms DESC
| LIMIT 10

-- Hourly error trend
FROM logs-*
| WHERE level == "ERROR"
| STATS count = COUNT(*) BY BUCKET(@timestamp, 1 hour)
| SORT `BUCKET(@timestamp, 1 hour)` ASC

-- Parse and analyze log message
FROM logs-*
| GROK message "%{TIMESTAMP_ISO8601:ts} %{LOGLEVEL:lvl} %{GREEDYDATA:msg}"
| WHERE lvl == "ERROR"
| STATS count = COUNT(*) BY msg
| SORT count DESC
| LIMIT 20
```
