## Elasticsearch Ingest Processors — Complete Reference

> Ingest processors are the building blocks of ingest pipelines.
> They transform, enrich, parse, and route documents BEFORE they are indexed.
> Think of them as a pre-indexing data transformation layer — like ETL inside Elasticsearch itself.

## 🔄 How Ingest Pipelines Work

```
Document arrives (PUT/POST/_bulk)
         │
         ▼
┌─────────────────────┐
│   Ingest Node        │   ← Pipeline runs here
│                      │
│  ┌──────────────┐   │
│  │ Processor 1  │   │   ← e.g., GROK (parse message)
│  └──────┬───────┘   │
│         ▼           │
│  ┌──────────────┐   │
│  │ Processor 2  │   │   ← e.g., DATE (parse timestamp)
│  └──────┬───────┘   │
│         ▼           │
│  ┌──────────────┐   │
│  │ Processor N  │   │   ← e.g., SET (add metadata)
│  └──────┬───────┘   │
│         ▼           │
│   on_failure: []    │   ← fallback processors if any fail
└─────────┬───────────┘
          │
          ▼
   Document indexed to Elasticsearch
```

## 🛠️ Pipeline Skeleton — Always Start With This

```json
PUT _ingest/pipeline/my-pipeline
{
  "description": "What this pipeline does",
  "processors": [
    {
      "processor_name": {
        "field": "source_field",
        "target_field": "destination_field",
        "ignore_failure": true,
        "if": "ctx.field != null",
        "tag": "my-processor-tag"
      }
    }
  ],
  "on_failure": [
    {
      "set": {
        "field": "error.message",
        "value": "{{ _ingest.on_failure_message }}"
      }
    }
  ]
}
```

**Common processor parameters (available on ALL processors):**

| Parameter | Purpose |
|---|---|
| `if` | Conditional — run only if Painless condition is true |
| `ignore_failure` | Don't fail pipeline if this processor errors |
| `on_failure` | Run these processors if this one fails |
| `tag` | Label for debugging |
| `description` | Human-readable note |

## 📋 All Ingest Processors — Complete List

### 🔤 1. Parsing & Extraction Processors

| Processor | Purpose | Doc |
|---|---|---|
| **GROK** | Parse unstructured text using named regex patterns | [📖](https://www.elastic.co/docs/reference/enrich-processor/grok-processor) |
| **DISSECT** | Parse text using delimiter patterns (faster than grok, no regex) | [📖](https://www.elastic.co/docs/reference/enrich-processor/dissect-processor) |
| **KV (Key-Value)** | Parse `key=value` pairs from strings | [📖](https://www.elastic.co/docs/reference/enrich-processor/kv-processor) |
| **CSV** | Parse CSV formatted strings into fields | [📖](https://www.elastic.co/docs/reference/enrich-processor/csv-processor) |
| **JSON** | Parse JSON string field into structured object | [📖](https://www.elastic.co/docs/reference/enrich-processor/json-processor) |
| **HTML Strip** | Remove HTML tags from field | [📖](https://www.elastic.co/docs/reference/enrich-processor/htmlstrip-processor) |
| **URL Decode** | URL-decode a string field | [📖](https://www.elastic.co/docs/reference/enrich-processor/urldecode-processor) |
| **Bytes** | Convert human-readable byte size to bytes (e.g., "1kb" → 1024) | [📖](https://www.elastic.co/docs/reference/enrich-processor/bytes-processor) |
| **Convert** | Convert field type (string → int, float, boolean, etc.) | [📖](https://www.elastic.co/docs/reference/enrich-processor/convert-processor) |

```json
// GROK example — parse Apache log
{
  "grok": {
    "field": "message",
    "patterns": ["%{COMBINEDAPACHELOG}"],
    "ignore_failure": true
  }
}

// DISSECT example — parse structured log
{
  "dissect": {
    "field": "message",
    "pattern": "%{timestamp} %{level} %{service} %{msg}"
  }
}

// KV example — parse "a=1 b=2 c=3"
{
  "kv": {
    "field": "message",
    "field_split": " ",
    "value_split": "="
  }
}
```

| Processor | Purpose | Doc |

```json
}
```

| **Trim** | Remove leading/trailing whitespace | [📖](https://www.elastic.co/docs/reference/enrich-processor/trim-processor) |
| **Uppercase** | Convert field value to uppercase | [📖](https://www.elastic.co/docs/reference/enrich-processor/uppercase-processor) |
| **Lowercase** | Convert field value to lowercase | [📖](https://www.elastic.co/docs/reference/enrich-processor/lowercase-processor) |

```json
// SET with dynamic value
{
  "set": {
    "field": "event.ingested",
    "value": "{{{_ingest.timestamp}}}"
  }
}

// SET with copy_from (copy field value)
{
  "set": {
    "field": "host.name",
    "copy_from": "agent.hostname",
    "ignore_empty_value": true
  }
}

// REMOVE multiple fields
{
  "remove": {
    "field": ["raw_message", "temp_field", "debug_info"],
    "ignore_missing": true
  }
}
```

### 🔀 6. Control Flow & Logic Processors

| Processor | Purpose | Doc |
|---|---|---|
| **Pipeline** | Call another pipeline from within a pipeline | [📖](https://www.elastic.co/docs/reference/enrich-processor/pipeline-processor) |
| **Foreach** | Apply a processor to each element of an array | [📖](https://www.elastic.co/docs/reference/enrich-processor/foreach-processor) |
| **Conditional (if)** | Run processor only if condition is true (not a processor itself — use `if` param on any processor) | Built-in |
| **Drop** | Drop the document entirely — don't index it | [📖](https://www.elastic.co/docs/reference/enrich-processor/drop-processor) |
| **Fail** | Raise an error with custom message (for validation) | [📖](https://www.elastic.co/docs/reference/enrich-processor/fail-processor) |
| **Reroute** | Change target index/data stream of the document | [📖](https://www.elastic.co/docs/reference/enrich-processor/reroute-processor) |
| **Terminate** | Stop pipeline processing and index as-is | [📖](https://www.elastic.co/docs/reference/enrich-processor/terminate-processor) |

```json
// PIPELINE — call sub-pipeline
{
  "pipeline": {
    "name": "common-enrichment-pipeline",
    "ignore_missing_pipeline": true
  }
}

// FOREACH — process each element in array
{
  "foreach": {
    "field": "dns.answers",
    "processor": {
      "uppercase": {
        "field": "_ingest._value.type"
      }
    }
  }
}

// DROP — don't index health check logs
{
  "drop": {
    "if": "ctx.http?.request?.url?.path == '/health'"
  }
}

// REROUTE — send to different data stream based on field
{
  "reroute": {
    "dataset": "{{service.name}}",
    "namespace": "production"
  }
}
```

### ⚙️ 10. Scripting Processor

| Processor | Purpose | Doc |
|---|---|---|
| **Script** | Run Painless script for custom logic not possible with other processors | [📖](https://www.elastic.co/docs/reference/enrich-processor/script-processor) |

```json
// SCRIPT — custom logic
{
  "script": {
    "lang": "painless",
    "source": """
      if (ctx.http?.response?.status_code != null) {
        int code = Integer.parseInt(ctx.http.response.status_code.toString());
        if (code >= 500) {
          ctx.event.outcome = 'failure';
        } else if (code >= 400) {
          ctx.event.outcome = 'client_error';
        } else {
          ctx.event.outcome = 'success';
        }
      }
    """
  }
}
```

## 🏗️ Real-World Pipeline — Full Example

```json
PUT _ingest/pipeline/nginx-access-logs
{
  "description": "Parse and enrich Nginx access logs",
  "processors": [
    {
      "grok": {
        "field": "message",
        "patterns": ["%{COMBINEDAPACHELOG}"],
        "ignore_failure": false,
        "tag": "parse-nginx-log"
      }
    },
    {
      "date": {
        "field": "timestamp",
        "target_field": "@timestamp",
        "formats": ["dd/MMM/yyyy:HH:mm:ss Z"],
        "tag": "parse-timestamp"
      }
    },
    {
      "convert": {
        "field": "response",
        "target_field": "http.response.status_code",
        "type": "integer",
        "ignore_missing": true
      }
    },
    {
      "convert": {
        "field": "bytes",
        "target_field": "http.response.body.bytes",
        "type": "long",
        "ignore_missing": true
      }
    },
    {
      "geoip": {
        "field": "clientip",
        "target_field": "source.geo",
        "ignore_missing": true
      }
    },
    {
      "user_agent": {
        "field": "agent",
        "target_field": "user_agent",
        "ignore_missing": true
      }
    },
    {
      "set": {
        "field": "event.ingested",
        "value": "{{{_ingest.timestamp}}}"
      }
    },
    {
      "drop": {
        "if": "ctx.request == '/health' || ctx.request == '/ping'"
      }
    },
    {
      "remove": {
        "field": ["message", "timestamp", "clientip", "response", "bytes", "agent"],
        "ignore_missing": true
      }
    }
  ],
  "on_failure": [
    {
      "set": {
        "field": "event.outcome",
        "value": "failure"
      }
    },
    {
      "set": {
        "field": "error.message",
        "value": "{{ _ingest.on_failure_message }}"
      }
    },
    {
      "set": {
        "field": "error.processor_tag",
        "value": "{{ _ingest.on_failure_processor_tag }}"
      }
    }
  ]
}
```

## ⚡ Quick Reference — Which Processor for What

| Task | Processor |
|---|---|
| Parse unstructured log line | `grok` |
| Parse structured log with delimiters | `dissect` |
| Parse `key=value` pairs | `kv` |
| Parse JSON string field | `json` |
| Parse timestamp string | `date` |
| Add geo location from IP | `geoip` / `ip_location` |
| Parse browser/OS from User-Agent | `user_agent` |
| Add fixed or computed field | `set` |
| Delete unwanted fields | `remove` |
| Change field type | `convert` |
| Don't index this document | `drop` |
| Call another pipeline | `pipeline` |
| Process array elements | `foreach` |
| Custom logic | `script` (Painless) |
| Run ML model | `inference` |
| Mask PII | `redact` |
| Route to different index | `reroute` |
| Create document fingerprint/ID | `fingerprint` |

## 💡 Key Tips

```
1. Always test with _simulate before applying to production
2. Use ignore_failure: true on non-critical processors
3. Always add on_failure at pipeline level to capture errors
4. Use if condition to skip processors when fields are missing
5. grok is powerful but slow — use dissect when format is fixed
6. Remove raw/temporary fields at the end to save storage
7. Date processor MUST run before any time-based routing
8. Fingerprint processor can generate _id to prevent duplicates
9. Reroute processor replaces the need for multiple pipelines
10. Use tag parameter for easy debugging in on_failure messages
```
